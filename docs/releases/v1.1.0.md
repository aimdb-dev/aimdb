# AimDB v1.1.0 Release Notes

**Stage profiling, `no_std` transforms, context-aware connectors and a safer MCP server!**

> Typed dataplane for distributed systems

---

## 🎯 Highlights

This release delivers three major capabilities across the full platform stack: **automatic per-stage performance profiling** (with LLM-powered bottleneck detection via MCP), **`no_std` support for the complete Transform API** (`.transform()` and `.transform_join()` now run on Embassy and WASM), and **context-aware serializers/deserializers** that give connector callbacks access to the runtime clock and logger. The MCP server gains a **public/read-only mode** with SSRF protection for internet-facing deployments, and a first community contribution ships in the form of a **`hello-mailbox` example**.

---

## ✨ Major Features

### 📊 Automatic Stage Profiling (Issue #58, RFC 014)

**Zero-instrumentation wall-clock timing for every `.source()`, `.tap()`, and `.link()` stage!**

Enable with `--features profiling` — zero overhead when the feature is off.

```rust
// Name stages for readable profiling output
builder.configure::<Temperature>(AppKey::Temp, |reg| {
    reg.buffer(BufferCfg::SingleLatest)
       .source(produce_temperature)
       .with_name("temperature_sensor")      // ← names the source stage
       .tap(log_temperature)
       .with_name("temperature_logger")      // ← names the tap stage
});
```

```bash
# Query via MCP
mcp_aimdb-local_get_stage_profiling(record_key: "Temp")
# → bottleneck: "slow_status_processor" (avg 103 ms)
# → recommendation: "Consider offloading or batching"
```

**Details:**
- Measures interval between successive `produce()` calls (source) and successive `recv()` yields (tap/link) — no per-value handler changes required
- Per-stage counters: `call_count`, `avg_time_ns`, `min_time_ns`, `max_time_ns`
- `RecordRegistrar::with_name("...")` always callable — no-op when profiling is disabled
- Works on `no_std + alloc` via `portable-atomic` with `fallback` + `critical-section` for 64-bit atomics on `thumbv7em-none-eabihf`
- Two new MCP tools: `get_stage_profiling` (bottleneck detection + human-readable recommendation) and `reset_stage_profiling`
- `remote-access-demo` updated: `SystemStatus.slow_status_processor` sleeps 100 ms deliberately so the bottleneck is immediately visible when you run the demo

### 🔄 `no_std` Transform API (Design 027)

**`.transform()` and `.transform_join()` now work on every runtime — Tokio, Embassy, and WASM!**

Multi-input join fan-in is no longer hardcoded to `tokio::sync::mpsc`. The new `JoinFanInRuntime` trait in `aimdb-executor` is implemented per adapter:

| Adapter | Backend | Capacity |
|---------|---------|----------|
| Tokio | `tokio::sync::mpsc::channel` | 64 |
| Embassy | `embassy_sync::Channel<CriticalSectionRawMutex, T, 8>` | 8 |
| WASM | `futures_channel::mpsc::channel` | 64 |

**New task-model join handler** — replaces the old callback form with a long-running task that owns the event loop:

```rust
// New (v1.1.0) — task model, no per-event heap allocation
builder.configure::<DewPoint>(AppKey::DewPoint, |reg| {
    reg.buffer(BufferCfg::SingleLatest)
       .transform_join(|join| {
           join.input::<Temperature>(AppKey::Temp)
               .input::<Humidity>(AppKey::Humidity)
               .on_triggers(dew_point_task)
       })
});

async fn dew_point_task(
    mut rx: JoinEventRx,
    producer: Producer<DewPoint, TokioAdapter>,
) {
    let mut temp = None;
    let mut humidity = None;
    while let Ok(trigger) = rx.recv().await {
        match trigger.index {
            0 => temp = Some(trigger.value::<Temperature>()),
            1 => humidity = Some(trigger.value::<Humidity>()),
            _ => {}
        }
        if let (Some(t), Some(h)) = (&temp, &humidity) {
            producer.produce(calculate_dew_point(t, h)).await;
        }
    }
}
```

**Breaking change** vs. the old `with_state().on_trigger(...)` callback form — see [Migration Guide](#migration-guide) below.

The weather-mesh demo stations now derive a `DewPoint` record from `Temperature` and `Humidity` via `transform_join` on both Tokio and Embassy.

### 🔌 Context-Aware Connectors (Design 026)

**Deserializers and serializers can now access the runtime clock and logger!**

```rust
// New: context-aware deserializer
connector.link_from("mqtt://broker/sensor/temperature")
    .with_deserializer(|ctx, bytes| {
        let ts = ctx.now();   // platform-independent timestamp
        ctx.log_debug(&format!("Received {} bytes at {:?}", bytes.len(), ts));
        serde_json::from_slice::<Temperature>(bytes).map_err(|e| e.to_string())
    });

// Old (still available for simple cases)
connector.link_from("mqtt://broker/sensor/temperature")
    .with_deserializer_raw(|bytes| {
        serde_json::from_slice::<Temperature>(bytes).map_err(|e| e.to_string())
    });
```

Symmetric `.with_serializer(|ctx, value| ...)` and `.with_serializer_raw(|value| ...)` APIs for outbound connectors. All existing connectors (MQTT, KNX, WebSocket) updated to propagate context.

### ✅ Writer-Exclusivity Validation (Issue #89)

`.source()`, `.transform()`, and `.link_from()` are now **mutually exclusive** — combining any two panics at configuration time with a clear error message instead of silently racing on the buffer:

```
thread 'main' panicked at 'Record "sensor::Temperature" already has a
source/transform and cannot also have a link_from "mqtt://...". A record
can only have one writer.'
```

Fan-in with multiple `.link_from()` connectors remains allowed.

## 🔧 Other Changes

### MCP Server

- **Public mode** (`--public` flag): Restricts the server to three read-only tools (`discover_instances`, `list_records`, `get_record`) for safe internet-facing deployments. Strips client-supplied `socket_path` arguments to prevent SSRF. Suppresses resources and prompts capabilities.
- **Default socket** (`--socket <PATH>`): Sets a default socket path at server startup. Resolution order: explicit argument → `--socket` flag → `AIMDB_SOCKET` env var. Reduces repetition when working with a single instance.

### New Examples

- **`hello-mailbox`** — minimal example demonstrating Mailbox (latest-wins) semantics using the synchronous API. First community contribution, by [@ggmaldo](https://github.com/ggmaldo).
- **Weather-mesh DewPoint** — all weather stations now derive `DewPoint` from `Temperature` + `Humidity` via `transform_join`, end-to-end on both Tokio and Embassy.

### Dependency Updates

- **`rand` 0.8 → 0.10.1** across workspace (`aimdb-data-contracts`, `aimdb-embassy-adapter`, examples). Migrated API: `gen` → `random`, `SmallRng` seed size 16 → 32, `RngExt` imports added.

## ⚠️ Breaking Changes

### Join handler API (`aimdb-core`, `aimdb-codegen`)

`JoinBuilder::with_state(...).on_trigger(Fn(...) -> Pin<Box<dyn Future>>)` is replaced by the task-model:

```rust
JoinBuilder::on_triggers(FnOnce(JoinEventRx, Producer<Output, R>) -> impl Future)
```

The handler now owns the full event loop, eliminating per-event heap allocation and allowing state to borrow across `.await` points. Generated code from `aimdb-codegen` is updated automatically.

### Connector serializer/deserializer APIs (`aimdb-core`)

| Old | New |
|-----|-----|
| `.with_deserializer(\|bytes\| ...)` | `.with_deserializer_raw(\|bytes\| ...)` |
| `.with_serializer(\|value\| ...)` | `.with_serializer_raw(\|value\| ...)` |

New context-aware forms: `.with_deserializer(\|ctx, bytes\| ...)` and `.with_serializer(\|ctx, value\| ...)`.

Internal types `InboundConnectorLink::deserializer`, `Router::route()`, `ConnectorLink::serializer`, and `OutboundRoute` have changed to use the `DeserializerKind` / `SerializerKind` enums. Custom connector implementations must update their router dispatch calls to pass `db.runtime_any()`.

### `TimeOps::duration_as_nanos` — new required method (`aimdb-executor`)

Implementations of the `TimeOps` trait must now provide `duration_as_nanos(duration: Self::Duration) -> u64`. Custom runtime adapters outside this workspace need to add this method; implementations should saturate rather than overflow.

## 🚀 Migration Guide

### Step 1: Update join handlers

```rust
// Before (callback model)
.transform_join(|join| {
    join.input::<Temperature>(AppKey::Temp)
        .input::<Humidity>(AppKey::Humidity)
        .with_state(MyState::default())
        .on_trigger(|trigger, state, producer| Box::pin(async move {
            // ...
        }))
})

// After (task model)
.transform_join(|join| {
    join.input::<Temperature>(AppKey::Temp)
        .input::<Humidity>(AppKey::Humidity)
        .on_triggers(my_join_task)
})

async fn my_join_task(
    mut rx: JoinEventRx,
    producer: Producer<DewPoint, TokioAdapter>,
) {
    // State lives here — no heap allocation per event
    let mut state = MyState::default();
    while let Ok(trigger) = rx.recv().await {
        // handle trigger
        producer.produce(compute(&state)).await;
    }
}
```

### Step 2: Update connector serializer/deserializer signatures

```rust
// Before
.with_deserializer(|bytes| serde_json::from_slice(bytes).map_err(|e| e.to_string()))
.with_serializer(|value| serde_json::to_vec(&value).map_err(|e| e.to_string()))

// After (plain — no context needed)
.with_deserializer_raw(|bytes| serde_json::from_slice(bytes).map_err(|e| e.to_string()))
.with_serializer_raw(|value| serde_json::to_vec(&value).map_err(|e| e.to_string()))

// Or: new context-aware form
.with_deserializer(|ctx, bytes| {
    // ctx.now(), ctx.log_debug(...)
    serde_json::from_slice(bytes).map_err(|e| e.to_string())
})
```

### Step 3: Custom `TimeOps` implementations

Add `duration_as_nanos` to any custom `TimeOps` impl:

```rust
fn duration_as_nanos(duration: Self::Duration) -> u64 {
    duration.as_nanos().min(u64::MAX as u128) as u64
}
```

### Step 4: Enable stage profiling (optional)

```toml
# Cargo.toml
[features]
profiling = ["aimdb-core/profiling", "aimdb-tokio-adapter/profiling"]
```

```bash
# Inspect bottlenecks via MCP
mcp_aimdb-local_get_stage_profiling(record_key: "")  # all records
mcp_aimdb-local_reset_stage_profiling(socket_path: "/tmp/myapp.sock")
```
